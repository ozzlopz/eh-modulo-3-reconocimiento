# Módulo 3 – Reconocimiento Pasivo y Activo

## Descripción

Este repositorio contiene el desarrollo de la **Tarea 3 del Módulo 3**.

El objetivo de la actividad fue comprender cómo se puede obtener información pública sobre sistemas y organizaciones y cómo esta información podría representar un riesgo si se expone de forma incorrecta.

---

## Contenido del proyecto

El repositorio incluye:

- **PDF con el desarrollo de la actividad**
- **Capturas de pantalla de las búsquedas realizadas**
- **Explicación de los resultados encontrados**
- **Mapa visual generado con Maltego**

---

## Parte 1 – Reconocimiento Pasivo con Google Dorks

En esta sección se realizaron **8 búsquedas utilizando Google Dorks**, que son operadores avanzados de búsqueda que permiten localizar información específica en internet.

Cada búsqueda fue documentada indicando:

- El dork utilizado
- Qué tipo de información puede revelar
- Si representa un riesgo de seguridad
- Evidencia mediante captura de pantalla

El objetivo fue comprender cómo la información pública puede ser encontrada fácilmente utilizando motores de búsqueda.

---

## Parte 2 – Google Hacking Database (GHDB)

Se investigaron **3 Google Dorks** dentro de la **Google Hacking Database (GHDB)** en Exploit Database.

Para cada dork se analizó:

- Qué tipo de información busca
- Qué tipo de exposición detecta
- La categoría de vulnerabilidad dentro de GHDB

Esta actividad permitió entender cómo se clasifican diferentes tipos de exposición de información en internet.

---

## Parte 3 – Análisis con Maltego

Se utilizó **Maltego** para visualizar la relación entre distintos elementos de infraestructura de internet.

El proceso fue el siguiente:

1. Crear un grafo nuevo
2. Agregar el dominio `maltego.com`
3. Ejecutar transformaciones para obtener:
   - DNS del dominio
   - Dirección IP asociada
   - Ubicación geográfica del servidor

Esto permitió visualizar cómo se conectan los diferentes componentes de una infraestructura en internet.

---

## Parte 4 – Reflexión

Se realizó una reflexión sobre:

- La diferencia entre **reconocimiento pasivo y activo**
- Por qué el reconocimiento sin autorización puede ser ilegal
- La importancia de proteger la información expuesta públicamente

Esta sección permitió comprender los aspectos **éticos y legales** del reconocimiento en ciberseguridad.
